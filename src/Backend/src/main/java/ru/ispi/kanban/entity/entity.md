### Зачем композитный ключ
* Уникальность связи — нельзя добавить одного пользователя в одну группу дважды.

* Отражает смысл данных — запись = связь A ↔ B, а не отдельный объект.

* Не нужен искусственный id — нет лишнего surrogate-ключа.

* Меньше индексов — PK уже индексирует нужные поля.

* Защита от дублей на уровне БД, а не кода.
