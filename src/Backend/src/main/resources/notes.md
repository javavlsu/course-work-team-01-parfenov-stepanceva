# Запись всяких заметок здесь:

### Описание структуры работы слоев
1. Контроллер - запросы к апи, всякие параметры и т.д.
2. Сервсис - получает что то от контроллера (Payload - полезная нагрузка, неизменяемая), начинает с этими данными работать, как то их сохранять менять или отдавать (В виде dto - как бы то что увидит пользователь, а не целый кусок из базы данных)
3. Репозиторий - имплиментирует интерфейс конкретной сущности и уже сам бегает в базу данных (не важно в какую)

### По поводу всяких конфигов
Есть замечательный файл application.yml - в нем можно поменять текущие настройки профиля
профиль dev - дефолтный, он просто скидывается в гит, там особо ничего не меняем, если надо делаем application-local.yml и внутри него меняем конфиг
#### Обратить внимание!!! Что если используется dev профиль, то в таком случае - хранилище это память при запуске, а не база данных в mysql !!!
